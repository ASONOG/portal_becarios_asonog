# Flujo de Usuario

## Actores
- **Administrador** — gestiona cuentas, asignaciones y revisión de documentos
- **Becario** — sube documentos según las asignaciones recibidas

## Flujo principal

### 1. Creación de cuenta
- El administrador crea la cuenta del becario desde el panel
- El becario no puede registrarse por su cuenta

### 2. Inicio de sesión
- El becario accede al sitio e inicia sesión con las credenciales proporcionadas

### 3. Revisión de asignaciones
- Una vez dentro, el becario ve las asignaciones activas creadas por el administrador
- Cada asignación indica qué documento debe subir

### 4. Subida de documento
- El becario sube el archivo correspondiente a cada asignación

### 5. Revisión por el administrador
- El administrador revisa el documento subido
- Puede **aprobarlo** o **rechazarlo**

### 6. Reenvío (si es rechazado)
- Si el documento es rechazado, el becario puede subir un nuevo archivo
- El ciclo se repite hasta que el documento sea aprobado
